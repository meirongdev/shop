# Shop Platform — 产品设计文档

> 版本：2.0 | 日期：2026-03-20 | 定位：2026 年云原生电商平台

---

## 一、产品定位

**Shop Platform** 是一个面向中小商家的 B2C 电商平台，核心差异点：

| 差异点 | 说明 |
|--------|------|
| **零门槛购物** | 游客无需注册/登录，即可浏览、下单、支付 |
| **积分生态** | 购物/签到/评价均可赚积分，积分可兑换商品或优惠券 |
| **互动游戏化** | 抢红包、砸金蛋、集卡、拼团等玩法驱动留存与传播 |
| **可扩展促销引擎** | 满减/折扣/买赠/秒杀等 13+ 种促销类型，运营零代码配置 |
| **商家赋能** | 卖家自助开店、上架商品、管理促销、查看数据 |
| **多端统一体验** | PC Web、移动 H5、开放 API 三端覆盖 |
| **智能个性化** | AI 推荐、个性化搜索（Phase 2 规划） |

## 二、用户角色

### 2.1 游客 (Guest)

无需任何注册，通过浏览器 Cookie Session 标识。

| 能力 | 说明 |
|------|------|
| 浏览商品 | 查看商品列表、详情、图片、评价 |
| 搜索商品 | 关键词搜索、分类筛选、价格排序 |
| 购物车 | 添加/删除商品，数量调整（buyer-bff Redis 持久化，登录后自动合并） |
| 下单 | 填写收货信息 + 邮箱，无需账号 |
| 支付 | 信用卡、PayPal（Apple/Google Pay 规划中） |
| 订单追踪 | 凭 order_token 查询订单状态 |
| 活动浏览 | 查看活动广场、游戏详情、奖池与历史说明 |
| 注册引导 | 登录后自动合并游客购物车；活动参与与积分能力要求登录 |

### 2.2 注册买家 (Registered Buyer)

| 能力类别 | 具体能力 |
|---------|---------|
| **账号** | 修改昵称、头像、绑定手机/邮箱、社交登录 |
| **地址簿** | 保存多个收货地址，设置默认 |
| **钱包** | 充值、余额支付、提现、绑定信用卡 |
| **积分** | 查看积分余额、流水、等级；签到赚积分 |
| **积分兑换** | 用积分兑换优惠券、实物礼品、虚拟奖品 |
| **会员等级** | SILVER → GOLD → PLATINUM，购物积分驱动升级 |
| **订单** | 历史订单、申请售后、退款、确认收货 |
| **优惠券** | 领取、使用、查看有效期 |
| **收藏夹** | 收藏商品和店铺，价格降低时收到提醒 |
| **评价** | 购买后评价（图文）、追评、买家秀 |
| **互动游戏** | 参与全部游戏类型，含抢红包、砸金蛋、集卡等 |

### 2.3 卖家 (Seller)

| 能力类别 | 具体能力 |
|---------|---------|
| **店铺** | 店铺资料、Logo、Banner、客服信息 |
| **商品** | 上架/下架、多规格 SKU、图片视频、库存维护 |
| **订单** | 查看/确认、发货（录入运单号）、处理退款 |
| **促销** | 创建满减/折扣/优惠券/买赠活动（可扩展引擎） |
| **活动** | 申请参与平台大促、秒杀活动报名 |
| **数据看板** | 销售额、访客数、转化率、复购率、积分发放量 |
| **结算** | 查看账单、申请提现 |

### 2.4 平台管理员 (Admin)

| 能力类别 | 具体能力 |
|---------|---------|
| **内容审核** | 商品审核、店铺申请审核 |
| **用户管理** | 封禁/解封账号、会员等级手动调整 |
| **促销配置** | 平台级满减、活动日历、促销类型扩展 |
| **活动管理** | 创建/上线/下线抢红包、砸金蛋等互动游戏 |
| **积分运营** | 积分规则配置（赚取倍率、到期策略） |
| **数据大屏** | 平台 GMV、用户增长、活动参与率、积分经济数据 |
| **风控** | 查看防作弊拦截日志、异常中奖率告警处理 |

## 三、核心功能模块

### 3.1 商品中心

#### 商品列表页
- 分类导航（面包屑 + 侧边树）
- 多维度筛选：价格区间、品牌、评分、标签、是否有促销
- 排序：综合、销量、价格升降、最新上架
- 瀑布流 / 网格切换
- 商品卡片：主图、名称、价格（划线原价）、销量、评分、促销标签、快速加购

#### 商品详情页
- 商品主图轮播（支持视频主图）
- SKU 选择矩阵（颜色 + 尺寸组合，自动匹配库存）
- 实时库存提示（"仅剩 3 件！"）
- 促销标签聚合展示（满减 + 优惠券 + 积分加倍活动）
- 积分预估：「购买此商品可获得约 XX 积分」
- 详情描述（富文本 + 图文混排）
- 买家评价（图片评价、追评、买家秀、评分分布）
- 相关推荐（协同过滤 / Phase 2 AI 推荐）
- 到货提醒（库存为零时订阅）

#### 搜索
- 全文搜索（Meilisearch，search-service 已实现；Phase 4 增强向量搜索）
- 搜索联想词、热门搜索榜
- 搜索结果高亮、筛选 + 排序联动
- 搜索历史（本地存储）

### 3.2 购物车

| 场景 | 实现 |
|------|------|
| 游客购物车 | buyer-bff Redis `buyer:guest:cart:{guestBuyerId}`，TTL 默认 48h（可配置） |
| 登录用户购物车 | 继续复用 order-service `cart_item` 持久化模型 |
| 登录后合并 | 登录成功后调用 `POST /buyer/v1/cart/merge` 逐项合并到账号购物车 |
| 库存校验 | 结账时实时校验，超库存自动降为可购数量 |
| 价格变动提示 | 商品价格变动时购物车显示提醒 |
| 失效商品 | 下架商品灰色标记，不参与结算 |
| 促销预估 | 购物车底部实时展示已触发的满减 / 即将触发的优惠 |
| 积分预估 | 「本单预计获得 XX 积分」 |

### 3.3 结账 & 支付

#### 游客结账流程
```
填写信息 → 选择支付方式 → 确认订单 → 支付 → 成功页（含 order_token）

必填：邮箱 + 收货地址
可选：优惠码
```

#### 注册用户结账流程
```
确认购物车 → 选择地址 → 选择优惠券 → 积分抵扣 → 选择支付方式 → 确认下单 → 支付
```

#### 积分抵扣（注册用户专属）
```
积分余额：1,250 pts
可用于抵扣：最多抵扣订单金额的 20%
使用积分：[   500   ] pts = -$5.00
```

#### 支付方式

| 方式 | 说明 | 状态 |
|------|------|------|
| 信用卡 / 借记卡 | Stripe | ✅ 已支持 |
| 平台钱包余额 | 内部扣减 | ✅ 已支持 |
| Apple Pay / Google Pay | Stripe Web | Phase 2 |
| PayPal | PayPal API | Phase 2 |
| 先买后付 (BNPL) | Klarna / Afterpay | Phase 2 |
| 微信支付 / 支付宝 | 跨境版 | Phase 3 |

### 3.4 订单管理

#### 订单状态机
```
PENDING_PAYMENT ──[30min超时]──→ CANCELLED
      │
      │[支付成功]
      ▼
    PAID ──[申请退款]──→ REFUND_REQUESTED ──→ REFUNDED
      │
      │[商家确认]
      ▼
  PROCESSING ──[发货]──→ SHIPPED ──[签收]──→ DELIVERED ──[7天]──→ COMPLETED
```

#### 游客订单追踪
- URL：`/buyer/order/track?token={order_token}`（无需登录）
- 显示：订单状态、物流轨迹时间线
- 操作：发起退款申请（需输入邮箱验证码）
- 引导：「注册账号后可永久查看历史订单」

### 3.5 积分与忠诚度体系

> 由 loyalty-service 驱动，积分是平台虚拟货币，独立于真实货币（钱包）。

#### 积分赚取途径

| 行为 | 积分规则 | 会员加成 |
|------|---------|---------|
| 购物消费 | 每消费 $1 得 10 积分 | GOLD ×1.5，PLATINUM ×2.0 |
| 每日签到 | 基础 5 积分 | — |
| 连签 7 天 | 第 7 天 25 积分 + 奖励券 | — |
| 完成评价 | 20 积分 / 次，每日上限 1 次 | — |
| 新用户注册 | 100 积分（一次性） | — |
| 邀请好友 | 好友首单后双方各得 50 积分 | — |
| 参与游戏中奖 | 游戏配置（积分红包等） | — |

#### 签到系统

```
签到日历（月视图）：
  ✓ ✓ ✗ ✓ ✓ ✓ ✓   ← 第 7 天：25分 + 奖励券！
  ✓ ✓ ✓ ✓ ✗ ...

连续签到奖励：
  Day 1-2:  5 分
  Day 3:    10 分（双倍）
  Day 4-6:  5 分
  Day 7:    25 分 + 随机优惠券

补签：每月最多 3 次，消耗 20 积分/次
```

#### 积分兑换目录（积分商城）

| 类型 | 示例 | 所需积分 |
|------|------|---------|
| 优惠券 | $10 无门槛券 | 500 pts |
| 优惠券 | 9 折券（最高减 $50） | 300 pts |
| 虚拟商品 | 视频会员月卡 | 1,000 pts |
| 实物礼品 | 品牌帆布袋 | 2,000 pts |
| 实物礼品 | 无线耳机 | 8,000 pts |
| 公益捐赠 | 捐 1 棵树（碳中和） | 100 pts |

#### 会员等级

| 等级 | 升级条件（年度积分） | 核心权益 |
|------|-------------------|---------|
| 🥈 SILVER | 注册即享 | 生日月 9.5 折、专属客服入口 |
| 🥇 GOLD | ≥ 5,000 pts/年 | 购物积分 ×1.5、免邮、优先发货 |
| 💎 PLATINUM | ≥ 20,000 pts/年 | 购物积分 ×2.0、提前参与大促、专属礼包 |

> 积分每年 3 月 31 日清零上年积分，会员等级按当年有效积分重新计算。

### 3.6 互动活动（小游戏）

> 由 activity-service 驱动，插件化引擎支持零代码新增游戏类型。

#### 游戏类型总览

| 类别 | 游戏 | 核心玩法 | 需要登录 |
|------|------|---------|---------|
| **即时抽奖** | 砸金蛋 | 点击破蛋，随机奖品（加权概率） | 否（每日1次） |
| **即时抽奖** | 大转盘 | 旋转指针，停格得奖 | 否（每日1次） |
| **即时抽奖** | 刮刮乐 | 刮开涂层，消耗积分换取 | 是 |
| **即时抽奖** | 九宫格 | 滚动高亮格，停格得奖 | 是 |
| **并发抢夺** | 抢红包 | 定时开抢，随机金额（积分），先到先得 | 是 |
| **社交裂变** | 砍价 | 邀请好友帮砍，砍到底免费得 | 是 |
| **社交裂变** | 拼团 | N 人成团享团购价，24h 未成团自动退款 | 是 |
| **社交裂变** | 集卡 / 集碎片 | 邀请好友各得一张，集齐兑大奖 | 是 |
| **长期养成** | 虚拟农场 | 购物 / 签到浇水，成熟领奖 | 是 |
| **知识竞技** | 答题赢奖 | 限时答题，答对得积分，连对有额外奖励 | 是 |

#### 游戏参与规则

- **入场条件**（运营配置）：免费 / 消耗积分 / 需购买满 $XX / 需特定会员等级
- **参与次数**：每日上限（运营配置，默认 1 次）
- **奖品类型**：积分、平台优惠券、实物礼品、虚拟商品、谢谢参与
- **防作弊**：IP 限速、设备指纹去重、账号年龄检验
- **后端决定结果**：所有游戏结果由服务端计算，前端只播放动效（防作弊）

#### 游戏入口

```
首页「活动广场」入口 → 活动列表页 → 游戏详情页 → 参与 → 结果页

游戏结果页：
  🎉 恭喜！获得「$10 优惠券」

  [立即使用]  [再玩一次（明日）]

  ─────────────────────────────
  🎰 还有更多活动等你来玩
  [查看所有活动]
```

#### 游客参与体验

```
游客进入活动广场：
  → 可浏览活动列表、详情页、奖池
  → 点击参与时跳转登录

登录后：
  → 参与游戏
  → 结果页展示 animationHint（抽奖停位 / 红包金额 / 集卡结果 / 农场进度）

  ← 游戏结果已保留 24h，注册后自动到账
```

### 3.7 促销 & 优惠

> 由 promotion-service 的可扩展引擎（Strategy 模式）驱动，运营通过 JSON 配置新增促销类型，无需改代码。

#### 促销类型矩阵

| 类型 | 示例 | 触发条件 | 权益 |
|------|------|---------|------|
| 满额减 | 满 $100 减 $10 | 订单金额 ≥ 阈值 | 固定金额减免 |
| 折扣 | 全场 9 折 | 任意订单 | 百分比减免 |
| 品类折扣 | 电子品类 8.5 折 | 含指定品类商品 | 品类专属折扣 |
| 买赠 | 买 2 送 1（最低价） | 同商品满件数 | 赠免费商品 |
| 满额包邮 | 订单满 $50 免运费 | 金额 ≥ 阈值 | 运费减免 |
| 限时秒杀 | 指定时间段内 8 折 | 时间窗口 | 折扣 |
| 新用户专享 | 首单减 $5 | 首次下单 | 固定减免 |
| 会员专属 | GOLD 会员额外 9.5 折 | 用户等级 | 叠加折扣 |
| 充值奖励 | 充值 $100 送 $10 券 | Kafka 事件驱动 | 发放优惠券 |
| 生日礼 | 生日月下单减 $8 | 生日月份 | 固定减免 |
| 积分加倍 | 活动期间购物积分 ×2 | 时间窗口 | 联动 loyalty-service |
| 商品组合 | A+B 同购减 $20 | 指定商品组合 | 固定减免 |
| 优惠码 | 输入 SAVE10 减 $10 | 用户输入 code | 固定/折扣 |

#### 叠加规则

```
EXCLUSIVE（互斥）：同时满足多个活动，只取折扣最大的一个
ADDITIVE（可叠加）：满减 + 优惠券可以同时使用
PRIORITY（优先级）：按优先级依次应用，总折扣有上限

结账页展示示例：
  原价      $200.00
  满减活动  -$20.00   「满$100减$20」
  优惠券    -$10.00   「10元无门槛券」
  积分抵扣  -$5.00    「500积分」
  ─────────────────
  合计      $165.00
  本单获得积分：+1,650 pts（GOLD 会员 ×1.5）
```

### 3.9 新用户注册福利

> 模块归属：profile-service 发布 `buyer.registered.v1` 事件，loyalty-service（积分+任务）和 promotion-service（优惠券）独立监听并行处理，无需新增服务。

#### 即时礼包（注册成功后 ≤ 5 秒到账）

| 奖励 | 内容 | 负责服务 |
|------|------|---------|
| 积分 | +100 pts，到账后可立即使用 | loyalty-service |
| 新人专属券 | $5 无门槛券，14 天有效期 | promotion-service |
| 免邮券 | 免运费 1 张，30 天有效期 | promotion-service |
| 折扣券 | 全场 9 折（最高减 $20），14 天有效期 | promotion-service |

#### 新人任务体系（14 天内完成，由 loyalty-service 管理）

| # | 任务 | 奖励 | 说明 |
|---|------|------|------|
| 1 | 完善头像和昵称 | +20 pts | 进入个人设置完成 |
| 2 | 绑定手机号 | +30 pts | 手机号 OTP 验证 |
| 3 | 完成首次签到 | +10 pts | 每日签到入口 |
| 4 | 首次加购商品 | +20 pts | 任意商品加入购物车 |
| 5 | 完成首单支付 | +100 pts + 额外 $3 券 | 核心转化任务 |
| 6 | 完成首次评价 | +30 pts | 首单签收后解锁 |
| 7 | 邀请第 1 位好友 | +50 pts | 好友注册即得 |
| — | **全部完成成就** | **额外 +100 pts** | 完成所有 7 项解锁 |

> 任务进度在个人中心「新人任务」卡片实时展示，14 天到期后未完成任务自动关闭。

#### 新用户专属区（marketplace-service 支持）

- 首页/列表页显示「新人专享价」标签（商品设 `new_user_price` 字段）
- 专属区仅首单有效；首单完成后自动下线

#### 邀请好友（裂变机制）

```
邀请人 → 分享注册链接（含 referrer_id）
好友注册（profile-service 写入 referral_source=REFERRAL）
→ 好友完成首单后 → loyalty-service 同时发放：
   邀请人：+50 pts
   被邀请人：+50 pts（叠加注册 100 pts）

防刷规则：每人每月最多邀请奖励 10 次；同设备/IP 去重；好友须完成首单才结算
```

## 四、游客购物体验设计

### 4.1 设计原则

> **零摩擦**：从进入网站到完成支付，全程不强制注册登录。
> **渐进引导**：在自然节点（结账成功、中奖、积分到账）引导注册，而非打断购物流程。

### 4.2 游客会话管理

```
首次访问：
  服务端生成 UUID session_id
  Cookie: session_id=xxx; Path=/; Max-Age=604800; SameSite=Lax; HttpOnly

游客购物车：
  Redis HSET cart:{session_id} {sku_id} {quantity}
  TTL: 7 天，每次操作自动续期

游客游戏参与：
  Redis SET game:{id}:guest:{session_id} 1 EX 86400
  （防重，游客每日限 1 次免费游戏）
```

## 八、数据分析

### 8.1 购物行为漏斗

```
访客 → 浏览商品 → 加入购物车 → 进入结账 → 完成支付
100%     60%          25%           15%          10%

优化目标：
  商品详情页 → 加购率 > 30%
  购物车     → 结账率 > 60%
  结账       → 支付率 > 70%（游客无注册墙是关键）
```

### 8.2 积分 & 忠诚度指标

```
积分经济健康度：
  积分发放量 / 积分消耗量       目标：1.1 ～ 1.3（轻微通胀，保持价值感）
  积分兑换率                    目标：> 40%（低说明积分无吸引力）
  签到 DAU / 总 DAU             目标：> 30%
  连签 7 天完成率               目标：> 15%

会员升级漏斗：
  SILVER → GOLD 转化率          目标：> 20%
  GOLD → PLATINUM 转化率        目标：> 8%
  PLATINUM 复购率                目标：> 70%
```

### 8.3 互动活动指标

```
游戏参与率   = 活动期间参与人数 / DAU     目标：> 15%
游戏拉新率   = 游戏引导注册用户数 / 注册总数
活动带动 GMV = 活动期间 GMV / 非活动期 GMV（同比）
奖励 ROI     = 活动带动 GMV / 奖励成本    目标：> 5×
拼团成团率   = 成功成团 / 发起总数        目标：> 60%
砍价完成率   = 砍到底 / 发起总数          目标：10%～20%（太高说明设置太宽松）
```

### 8.4 新用户转化指标

```
新用户礼包领取率   = 注册后进入 /welcome 页面 / 注册总数   目标：> 90%
新人首单转化率     = 注册后 7 天内完成首单 / 注册总数      目标：> 30%
任务完成率（全7项）= 全部完成任务的用户 / 注册总数         目标：> 10%
邀请裂变系数       = 受邀注册用户 / 月新增注册总数          目标：> 15%
新人券使用率       = 新人券使用 / 新人券发放               目标：> 40%
30 天留存率        = 注册后 30 天内有购买行为用户 / 注册数  目标：> 25%
邀请欺诈拦截率     = 拦截异常邀请 / 邀请总数               目标：< 2%
```

### 8.5 核心业务 KPI

| 指标 | 定义 | 目标 |
|------|------|------|
| GMV | 总成交额 | — |
| 转化率 | 下单用户 / 访客 | > 3% |
| 客单价 | GMV / 订单数 | — |
| 复购率 | 2+ 次购买用户占比 | > 35% |
| 游客下单率 | 游客下单 / 游客访问 | > 5% |
| 游客注册转化率 | 游客注册 / 游客下单 | > 25% |
| 退款率 | 退款订单 / 总订单 | < 5% |
| 促销 ROI | 促销带动 GMV / 促销成本 | > 5× |
| 积分活跃账户率 | 近 30 天有积分行为账户 / 总注册用户 | > 50% |
| 活动参与率 | 活动期间参与游戏人数 / DAU | > 15% |
